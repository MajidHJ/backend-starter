### 1. Domain Error vs HTTP Error

* A **Domain Error** represents a business rule violation (e.g., `UserNotFound`)
* An **HTTP Error** is just the protocol-level translation (e.g., 404)
* The domain layer must never depend on FastAPI or HTTP concepts
* Error translation should happen in the API layer (later via centralized handler)

---

### 2. Why `return None` / `False` Is Problematic

* `None` is ambiguous and hides the real reason for failure
* It forces route-level conditional logic (`if user is None`)
* It leads to inconsistent API behavior as the project grows
* Explicit exceptions are more expressive and maintainable

---

Defined expected domain-level errors:

* `UserNotFound`
* `EmailAlreadyExists`
* `InvalidUserData`
